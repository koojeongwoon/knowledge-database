# 프로젝트 규칙 (Rules)

## 1. 로컬 지식베이스 우선 참조 지침
- **모든 사용자의 질문 및 요청에 대해 답변을 작성하기 전에, 우선적으로 무조건 `retrieve_wiki_knowledge` 스킬을 실행**하여 로컬 지식베이스의 누적 기록을 조회하십시오.
- 관련 지식이 없어 임계치 아래로 판단되더라도 일단 항상 1회 조회하는 것을 원칙으로 하며, 조회된 결과가 있는 경우 이를 최우선으로 반영하여 답변을 작성해야 합니다.

## 2. Context7 (기술 문서 RAG) MCP 사용 지침
- **사용 시점**:
  - 라이브러리, 프레임워크, SDK, API, CLI 도구, 클라우드 서비스(React, Next.js, Spring Boot, Prisma, Tailwind 등)의 최신 문서를 참조해야 할 때.
  - API 문법, 마이그레이션 가이드, 환경 설정 정보 등 학습 데이터만으로는 부정확할 수 있는 기술적 스펙을 확인할 때.
- **금지 시점**:
  - 코드 리팩토링, 스크립트 작성, 비즈니스 로직 디버깅, 코드 리뷰.
  - 일반적인 프로그래밍 개념(예: OOP, 자료구조 개념 등) 질문.

## 3. Playwright (브라우저 자동화/렌더링) MCP 사용 지침
- **사용 시점**:
  - JavaScript 렌더링이 필수적인 동적 웹 페이지(SPA)의 데이터를 수집해야 할 때.
  - 웹 화면을 캡처(Screenshot)하여 레이아웃, 스타일, UI 에러 상태 등을 시각적으로 분석해야 할 때.
  - 버튼 클릭, 로그인, 폼 작성 등 웹페이지 내 사용자 상호작용(Interaction)이 동반되는 정보를 수집해야 할 때.
- **금지 시점**:
  - JavaScript 실행이 불필요한 단순 정적 웹페이지, API 문서, 텍스트 정보의 경우 플레이라이트 대신 더 가볍고 빠른 `read_url_content` 등의 텍스트 기반 도구를 우선 사용하십시오.
