# 프로젝트 규칙 (Rules)

## 1. 로컬 및 원격 MCP 지식베이스 우선 참조 지침
* **모든 사용자의 질문 및 요청에 대해 답변을 작성하기 전에, 우선적으로 무조건 `search_wiki_knowledge` MCP 도구를 실행**하여 지식베이스의 누적 기록을 조회하십시오.
* 관련 지식이 없어 임계치 아래로 판단되더라도 일단 항상 1회 조회하는 것을 원칙으로 하며, 조회된 결과가 있는 경우 이를 최우선으로 반영하여 답변을 작성해야 합니다.

## 2. RAG 도구와 R2 스토리지 동기화 지침
* 지식을 새로 작성(커밋)하거나 데이터베이스 인덱싱(`run_database_indexing`)을 실행할 때는 사용자가 대기하지 않도록 **비동기 백그라운드(서브에이전트)로 처리**하십시오.
* 본체 에이전트는 사용자에게 "지식화 및 인덱싱을 백그라운드에서 진행하겠습니다"라는 안내를 즉시 응답하여 대화 대기 시간을 최소화해야 합니다.
* 이후 백그라운드에서 `self` 또는 `research` 서브에이전트가 `commit_new_knowledge` 및 `run_database_indexing` MCP 도구를 호출하여 동기화 및 인덱싱 작업을 최종 완수하도록 조치합니다.

## 3. 지식화 시 세션 아티팩트(Artifacts) 이관 동의 확인 지침
* 모든 지식 영속화 및 커밋(`commit_new_knowledge` 사용)을 실행하기 전에, 현재 대화 세션에서 작성된 아티팩트(설계서, 보고서 등)가 존재하는지 반드시 확인하십시오.
* 아티팩트가 존재할 경우, 사용자에게 **"이전에 작성된 아티팩트(예: [파일명](링크))도 함께 영구 지식베이스에 복사하고 링크를 연결해 드릴까요?"**라고 물어보고 승인을 얻은 뒤 프로세스를 진행해야 합니다.
* 사용자가 동의하면 아티팩트의 경로를 `commit_new_knowledge` 도구의 `resource_paths`에 추가하여 이관을 보장하십시오.
